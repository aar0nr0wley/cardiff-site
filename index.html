Cardiff

A personal iOS health dashboard.

What it is

Cardiff is a single-user iOS app that consolidates personal health signals — lab results, vital signs, workouts, weight, and nutrition — into one timeline so trends are easier to see across categories.

It is built for personal use by the developer. There is no multi-user product, no marketplace, no advertising, and no third-party data sharing.

Health data Cardiff accesses

  • HealthKit standard data — heart rate samples recorded by Apple Watch during a workout (read-only).
  • HealthKit Clinical Health Records — lab results (HKClinicalTypeIdentifierLabResultRecord) and vital signs (HKClinicalTypeIdentifierVitalSignRecord) imported into iOS Health from the user's healthcare provider (Sutter Health, via Epic FHIR R4). Cardiff queries these records, parses the FHIR Observation resources locally, and displays the results in a labs timeline within the app.

All HealthKit access is read-only. Cardiff does not write back to HealthKit and does not request write permissions for clinical data.

How data is stored

Health records read from HealthKit are stored in the user's own Supabase Postgres instance. Each table enforces row-level security (auth.uid() = user_id), meaning only the authenticated owner of the data can read or modify their own rows. The Supabase instance is single-tenant (one user) and not exposed to any other party.

No clinical data is transmitted to any third-party analytics service, ad network, or external API. There is no telemetry, no event tracking, and no cross-device sharing.

Third-party services

  • Supabase — application database (Postgres) and authentication. The user owns the project.
  • Anthropic Claude API — used optionally for natural-language summaries of workout history. Only workout metadata (exercise name, set count, dates) is sent. Clinical health records are never sent to Claude.

Privacy summary

  • Read-only access to HealthKit data.
  • Data stored only in the user's own Supabase instance.
  • No third-party analytics, advertising, or tracking.
  • Clinical data never sent to external APIs (including Claude).
  • Data can be deleted at any time by the user from within the app or via Supabase.
